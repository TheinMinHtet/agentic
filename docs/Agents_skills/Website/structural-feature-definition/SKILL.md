## Overview
This skill provides a framework for breaking down product requirements into defined, structured features. It ensures that every feature is mapped to a user need, a business objective, and technical constraints.

## Usage
Use this when you have a general product vision or user needs, and need to translate them into a structured, actionable feature list.

## Steps
1.  **Requirement Gathering**: Collect user stories, feedback, and stakeholder goals.
2.  **Feature Categorization**: Group ideas into logical modules (e.g., Auth, Dashboard, Checkout).
3.  **Definition**: For each feature, define:
    *   **User Benefit**: What problem does it solve?
    *   **Scope**: What is included/excluded?
    *   **Acceptance Criteria**: What must be true for the feature to be considered done?
4.  **Prioritization**: Rank features using a framework (like MoSCoW or RICE).
5.  **Documentation**: Log results in a feature backlog.

## Pitfalls
*   **Feature Creep**: Including "nice-to-haves" that dilute the core value proposition.
*   **Ambiguous Requirements**: Failing to define what "done" looks like leads to rework.
*   **Ignoring Technical Debt**: Not balancing new features with architectural maintenance.
