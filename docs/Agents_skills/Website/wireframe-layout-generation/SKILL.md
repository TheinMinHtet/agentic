## Overview
This skill focuses on creating structural blueprints for digital products. It emphasizes layout, hierarchy, and information architecture over visual design, ensuring that user flows are logical before adding polish.

## Usage
Use this after defining features to determine how they should be laid out on the screen to optimize user experience.

## Steps
1.  **Define Information Hierarchy**: Determine the primary, secondary, and tertiary elements for a given view.
2.  **Sketch/Structure**: Define placement of components (navigation, content containers, CTAs).
3.  **Layout Patterns**: Select proven patterns (e.g., F-pattern for reading, card-based for feed).
4.  **Review Usability**: Evaluate layout efficiency for the intended user flow.
5.  **Iteration**: Refine based on feedback or constraints.

## Pitfalls
*   **Over-Designing**: Focusing on colors and fonts instead of layout and flow.
*   **Inconsistent Spacing**: Lack of a consistent grid system.
*   **Poor Accessibility**: Designing layouts that are difficult to navigate for keyboard or screen-reader users.
